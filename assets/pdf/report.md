# Matrix-Game 3.0: Real-Time and Streaming Interactive World Model with Long-Horizon Memory

Skywork AI / Matrix-Game Team

## 摘要

Matrix-Game 3.0 是一个面向交互式世界模型的技术框架，目标是在同一系统中同时实现：

- 长时程记忆一致性
- 720p 高分辨率生成
- 实时推理
- 精确动作控制

论文指出，现有方法通常难以同时兼顾“长时记忆一致性”和“高分辨率实时生成”。为了解决这个问题，Matrix-Game 3.0 在数据、模型和推理部署三方面联合设计：

1. **数据系统升级**：构建工业级“无限数据引擎”，融合 Unreal Engine 合成数据、AAA 游戏录制数据和真实世界视频数据。
2. **长时程一致性建模**：通过误差感知训练、自纠错机制、相机感知记忆检索与注入，提高长时段时空一致性。
3. **实时推理优化**：通过多段自回归蒸馏、模型量化和轻量化 VAE 解码，实现高效实时生成。

实验结果显示：

- 5B 模型在 720p 下可实现 **最高 40 FPS** 实时生成
- 能在 **分钟级序列** 中保持稳定的记忆一致性
- 扩展到 **2×14B / 28B 级别模型** 后，画质、动态表现和泛化能力进一步提升

## 1. 引言

世界模型旨在根据过去观测与用户动作预测未来环境状态，应用场景包括：

- 机器人规划与控制
- 游戏与娱乐
- XR / 沉浸式交互
- 具身智能与工业流程

论文认为，真正可用的交互式世界模型必须满足一个关键前提：

> **具备长时程时空一致性的实时生成能力**。

如果做不到这一点，系统要么退化成不连贯的短片段生成，要么延迟过高，无法部署。

论文回顾了已有工作的不足：

- 一些模型支持实时交互，但缺乏长期记忆能力
- 一些模型能延长视频长度，但难以保持实时性
- 一些闭源系统展现出较强能力，但细节未公开，难以复现

Matrix-Game 3.0 的目标，就是在开放研究框架下同时实现：

- 动作可控
- 长时记忆
- 高分辨率
- 实时交互

## 2. 相关工作

### 2.1 视频生成模型

当前主流视频生成模型大多基于 Diffusion Transformer（DiT）。

- 闭源模型如 Sora、Kling、Hailuo 在高质量视频生成方面取得进展
- 开源模型如 Wan、Magi-1、LTX-2.3 推动了可复现研究

但这些方法大多偏向 **离线视频生成**，对动作建模、交互控制和长时一致性支持不足。

### 2.2 长时程视频生成

长视频生成的核心问题包括：

- 误差累积
- 训练-推理分布不一致
- 自回归稳定性不足

相关方法如 Diffusion Forcing、Self-Forcing、Self-Forcing++、Causal Forcing 等，主要从训练策略和生成机制上缓解上述问题，但普遍缺乏显式记忆机制。

### 2.3 交互式世界模型

交互式世界模型比普通视频生成更难，因为它需要同时满足：

- 动作条件控制
- 长时一致性
- 实时推理

已有工作如 Genie、Genie-2、Genie-3、Matrix-Game 2.0、WorldPlay 等不断推进该方向，但论文认为仍然缺少一个统一框架来兼顾所有关键能力。

## 3. 方法

Matrix-Game 3.0 包含四个核心组件：

1. **误差感知交互式基础模型**
2. **长时程记忆机制**
3. **训练-推理对齐的少步蒸馏**
4. **实时推理加速模块**

### 3.1 误差感知交互式基础模型

基础模型以动作引导、自纠错的双向 DiT 为核心。

#### 关键原则

1. **架构一致性**  
   基础模型与蒸馏模型采用统一的双向结构，避免 teacher-student 架构差异带来的不稳定。

2. **对不完美上下文的鲁棒性**  
   推理时历史帧来自模型自身生成，必然带有误差，因此训练时也要让模型见到带误差的历史条件。

#### 动作注入方式

- 键盘等离散动作：通过 Cross-Attention 注入
- 鼠标等连续控制信号：通过 Self-Attention 注入

这样可以实现更精确稳定的交互控制。

#### 误差缓冲与自纠错

模型在训练时维护一个误差缓冲区：

- 先计算当前预测与真实 latent 之间的残差
- 将残差存入 error buffer
- 再从 error buffer 采样，将误差重新注入历史 latent

这样训练出的模型更接近真实推理环境，有助于：

- 降低曝光偏差
- 增强长时推理鲁棒性
- 为后续蒸馏提供更稳定的教师行为

### 3.2 长时程记忆

论文指出，仅依赖最近历史帧不足以保证长时一致性，模型还需要访问更久远的记忆观测。

#### 设计选择

作者评估了两类思路：

1. **隐式稀疏长上下文建模**  
   概念上优雅，但在高噪声阶段，记忆选择不稳定，难以收敛。

2. **显式相机感知记忆建模**  
   更稳定，但如果额外设计独立记忆分支，可能导致特征不对齐、收敛变慢。

#### 最终方案

Matrix-Game 3.0 采用统一的 DiT 框架，将以下内容放入同一注意力空间中：

- 检索得到的记忆 latent
- 最近历史帧 latent
- 当前要预测的 noisy latent

这样模型可以在同一层级中联合建模：

- 短期连续性
- 长期记忆锚点
- 当前预测目标

#### 相机感知记忆检索

不是所有历史观测都适合当前视角。系统根据：

- 相机位姿
- 视场重叠

来检索最相关的历史记忆，并加入相对 Plücker 几何编码，帮助模型理解不同视角下同一场景如何对齐。

#### 记忆路径的训练-推理对齐

训练时的记忆和历史 latent 来自真实数据，推理时则来自模型生成，因此同样会有分布偏差。作者将误差注入扩展到整个条件上下文，包括：

- 记忆 latent
- 历史 latent

从而提升模型在真实长时 rollout 中的稳定性。

#### 时间编码增强

作者还增强了时间 RoPE：

- 不仅编码当前片段内相对位置
- 还编码每个记忆 / 历史 / 预测帧在全序列中的原始时间索引

同时引入 **head-wise 扰动 RoPE base**，减轻远距离记忆与当前帧之间的周期性位置混叠问题。

### 3.3 训练-推理对齐的少步蒸馏

传统蒸馏方法往往采用因果 student，并以 chunk 为单位进行推理；但在本文的双向自回归设定下，student 的推理跨度与 teacher 对齐，因此需要新的训练策略。

#### 核心思想

采用 **多段自生成推理（multi-segment self-generated inference）**：

- student 连续 rollout 多个 segment
- 当前 segment 的 past frame 来自前一个 segment 的尾部
- memory 从在线更新的 memory pool 中按视角检索
- 第一个 segment 则退化成 I2V 模式

训练时随机停止在某一段，将结果送入 teacher 和 critic，通过 **Distribution Matching Distillation (DMD)** 优化 student。

目标是让 student 在训练阶段就模拟真实 few-step 推理过程，从而实现训练-推理一致。

### 3.4 实时推理

为了让系统达到工业级实时生成速度，作者引入了多种加速策略：

- **DiT INT8 量化**：仅对 attention projection 层量化
- **MG-LightVAE**：轻量化剪枝 VAE 解码器
- **GPU 记忆检索**：将相机感知记忆检索放到 GPU 执行

#### 部署方式

异步部署方案中：

- 8 张 GPU 用于 DiT 推理
- 1 张 GPU 用于 VAE 解码

最终整体系统可实现 **最高 40 FPS**。

#### 加速消融结果

在 75% VAE 剪枝配置下，论文给出如下对比：

| 配置 | FPS | 相比完整配置下降 |
|---|---:|---:|
| Full | ~40 | - |
| 去掉 INT8 quantization | 27.38 | 12.62 |
| 去掉 MG-LightVAE | 25.79 | 14.21 |
| 去掉 GPU retrieval | 6.60 | 33.40 |

结论：

- 所有优化都有贡献
- **GPU retrieval 是最关键的加速组件**
- 最终 40 FPS 依赖多个优化共同作用，而非单一技巧

### 3.5 大模型扩展

作者进一步将系统扩展到 **MoE-28B** 级别，以提升：

- 泛化能力
- 动态建模能力
- 长时一致性

关键策略包括：

- 将动作控制能力与视觉细节建模解耦到不同噪声阶段
- 高噪声模型使用动作精确数据训练
- 低噪声模型引入互联网视频增强泛化
- 渐进式提升分辨率与视频长度
- 对第一人称 / 第三人称分别训练高噪声模型，共享低噪声模型

这样既保留控制能力，也提升了大模型对长视频和复杂场景的表达能力。

## 4. 数据系统

论文构建了一个统一的数据系统，用于大规模高质量世界模型训练，主要融合三类来源：

1. Unreal Engine 合成数据
2. AAA 游戏录制数据
3. 真实世界视频数据

### 4.1 Unreal Engine 数据生产

作者构建了 Unreal-Gen 管线，特点包括：

- 使用 1000+ 自建 UE5 场景
- 基于 Nanite 与 Lumen，实现电影级渲染
- 在同一 engine tick 内同步采集：
  - RGB 图像
  - 玩家位姿
  - 相机 6DoF 位姿
  - 动作向量

其核心优势是 **tick 级同步**，保证视觉帧与动作/位姿之间零时序误差。

#### 导航与探索

采用 NavMesh + 强化学习混合代理：

- 高层 RL 负责选择导航目标
- 低层 NavMesh 负责路径规划
- 配合卡住检测、多级 fallback 策略，保证自动探索稳定执行

#### 相机与角色多样性

- 相机 yaw / pitch 随机化
- 支持第一人称与第三人称视角
- 角色由可组合部件构成（衣物、鞋子、头发、帽子、配饰等）
- 理论组合数超过 **10^8**

#### 工业自动化

系统支持：

- 多实例并发渲染
- 云端同步
- 自动质检
- webhook 闭环监控

### 4.2 AAA 游戏数据录制系统

作者设计了统一的四层解耦架构，支持多款 AAA 游戏同步采集，包括：

- GTA V
- Red Dead Redemption 2
- Palworld
- Cyberpunk 2077
- Hogwarts Legacy

四层结构包括：

1. 游戏进程层
2. Agent 控制层
3. 录制协调层
4. 数据输出层

系统还能通过物理方式自动推断 WSAD 控制方向，减少人工标注偏差，整体数据准确率超过 **99%**。

### 4.3 真实世界数据

为补充合成数据在光照变化和自然相机轨迹方面的不足，论文引入四类真实世界数据集：

- DL3DV-10K
- RealEstate10K
- OmniWorld
- SpatialVid

覆盖场景包括：

- 室内
- 城市街景
- 户外地标
- 无人机视角
- 车载视角

作者还统一使用 ViPE 对真实世界数据重新标注位姿，以消除不同数据源之间的坐标系不一致问题。

### 4.4 数据标注与质量过滤

#### 文本标注

作者使用 InternVL3.5-8B 生成层次化描述，包括：

1. Narrative caption
2. Static scene caption
3. Dense temporal caption
4. 感知质量评分

#### 质量过滤

从几何一致性、全局运动异常、相机速度等角度进行过滤，仅保留满足全部条件的数据轨迹。结合感知质量过滤后，剔除了约 **20%** 原始数据，得到高质量训练集。

## 5. 实验

### 5.1 设置

#### 交互式基础模型

- 基于 Wan2.2-TI2V-5B 架构
- 动作模块接入前 15 个 DiT block
- 训练中大多数情况下使用 4 个 past frame + 10 个 current noisy latent
- 其余情况退化为 action-conditioned I2V

#### 记忆增强基础模型

- 在基础模型之上初始化
- 联合输入：5 个 memory latent + 4 个 past latent + 10 个 noisy latent
- 使用约 4.8M 视频片段训练

#### 蒸馏模型

- teacher、critic、student 均由记忆增强基础模型初始化
- 先单段冷启动，再切换到多段自生成推理
- segment 数在 1 到 6 之间随机采样

### 5.2 结果与分析

#### 基础模型结果

论文展示了交互式基础模型具备：

- 基本动作可控性
- 背景稳定、无明显漂移
- 相机移动时合理的缩放与视角变化

#### 记忆增强模型结果

在“场景回访”实验中，后半段动作会反向执行，使相机回到先前访问过的位置。结果显示模型能够恢复：

- 之前出现过的局部几何结构
- 物体配置
- 建筑立面纹理
- 场景细节

这说明模型确实在使用长期记忆，而不是仅依赖短期连续性。

#### 大模型结果

28B 模型在第三人称长视频生成中表现出：

- 更强的时间一致性
- 更稳定的角色与场景身份保持
- 更丰富的动态与光照变化
- 更好的复杂环境泛化能力

#### 蒸馏模型结果

蒸馏后的 few-step 模型继承了基础模型的长期记忆能力，能够：

- 在后续帧中恢复被遮挡过的内容
- 在新出现的场景中保持风格与内容稳定
- 在长序列末端无明显漂移

#### VAE 剪枝结果

论文给出 MG-LightVAE 与原始 Wan2.2 VAE 的重建质量与速度对比：

| 模型 | PSNR | SSIM | Full(s) | Dec.(s) |
|---|---:|---:|---:|---:|
| Wan2.2 VAE | 33.79 | 0.99 | 0.99 | 0.76 |
| MG-LightVAE (50%) | 31.84 | 0.99 | 0.52 | 0.30 |
| MG-LightVAE (75%) | 31.14 | 0.99 | 0.35 | 0.13 |

结论：

- 50% 剪枝在画质下降有限的前提下显著加速
- 75% 剪枝进一步提速，但画质下降更明显

## 6. 结论

Matrix-Game 3.0 提出了一个统一框架，用于同时解决：

- 长时记忆一致性
- 高分辨率生成
- 实时推理
- 动作精确控制

其核心贡献可以概括为：

1. **工业级数据引擎**：融合合成、AAA 游戏与真实数据
2. **误差感知交互基础模型**：增强长时自回归稳定性
3. **相机感知记忆机制**：改善跨时间、跨视角一致性
4. **训练-推理对齐蒸馏**：实现 few-step 实时推理
5. **系统级加速优化**：使 5B 模型在 720p 下达到 40 FPS

论文最后提出未来方向：

- 继续扩大模型和数据规模
- 设计更高效的高分辨率长序列架构
- 研究更高级的长期记忆机制
